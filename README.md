# dnd-agent

An AI Dungeon Master that runs a prewritten D&D 5e campaign. The LLM is the actor
(narration, NPCs, adjudication), a SQLite database is the canonical game state, and a
dice engine is the referee — so the model never has to "remember" numbers it can get wrong.

This is a monorepo with two parts:

| Path                | What it is                                                                 |
|---------------------|----------------------------------------------------------------------------|
| `dnd-dm/`           | **The live backend.** Python engine + Anthropic-powered DM brain, served over FastAPI. This is what the Mac mini runs. |
| `DndAgent/frontend/`| **The web UI** (Next.js / "A.R.C.A.N.A."). The table interface players use. |
| `DndAgent/backend/` | A legacy/alternate backend (Gemini/OpenAI). Kept for history; **not** used by the Mac deployment. |
| `DndAgent/evaluation.ipynb` | Project evaluation notebook. |

## Deploy on the always-on Mac mini

The Mac runs `dnd-dm` (backend, port 8000) + `DndAgent/frontend` (UI, port 3000),
both as launchd services that auto-start at login and restart on crash, reachable over
Tailscale. One script does the platform-specific setup:

```bash
git clone <this-repo-url> dnd-agent
cd dnd-agent
bash dnd-dm/deploy/setup-macos.sh
```

Prerequisites on the Mac (Homebrew): `python@3.12`, `node`, `tailscale` (logged in).
The script rebuilds the Python venv and `node_modules`, points the UI at the Mac's
Tailscale IP, and installs the launchd services. After it finishes, play at
`http://<mac-tailscale-ip>:3000/table` from any device on your tailnet.

## Secrets

No keys are committed. Copy the templates and fill them in:

```bash
cp dnd-dm/.env.example dnd-dm/.env      # ANTHROPIC_API_KEY=sk-ant-...
```

`.env` / `.env.local` are gitignored. The frontend's API URL is generated by the setup
script; set `DndAgent/frontend/.env.local` by hand only for non-Tailscale hosts.

## Local (Linux/dev)

```bash
cd dnd-dm
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
cp .env.example .env   # add your Anthropic key
.venv/bin/uvicorn web.adapter:app --host 0.0.0.0 --port 8000
```
